# RocketCom 航電系統測試環境與測試方法說明書

本文件詳細記錄了成大 ISP 航電組在 **STM32F407VGT6** 主控板上進行的軟韌體測試環境配置、硬體連接、各模組的測試方法以及驗證步驟，作為後續整合測試與發射準備的黃金參考標準。

---

## 📅 測試環境與基本規格

### 1. 硬體規格與核心晶片
* **核心主控**：STM32F407VGT6 (LQFP100)
* **板載感測器**：
  * 高-G 加速度計：`ADXL375` (±200g, SPI1)
  * 氣壓高度計：`BMP388` (厘米級相對高度, SPI1)
  * 標準姿態 IMU：`BMI088` (±24g 加速度 & ±2000°/s 角速度, SPI2)
* **板載儲存外設**：
  * SPI Flash：`W25Q128` (128M-bit / 16MB, SPI3)
  * SD 卡槽：MicroSD 卡槽 (SDIO 4-bit 模式)

### 2. 軟體與開發環境
* **開發工具**：STM32CubeIDE + STM32CubeMX
* **編譯工具鏈**：`arm-none-eabi-gcc` v16.1.0 / `arm-none-eabi-binutils` v2.46.0
* **作業系統**：FreeRTOS (CMSIS_V2)
* **文件系統**：FatFs 整合 (用於 SD 卡數據記錄)

## 🛠️ 一、 系統基礎測試環境與方法

### 1. LED 系統心跳指示燈測試
* **硬體引腳**：`PE2` (Net Label: `LED_SYS`)，控制邏輯為低電平點亮，高電平熄滅。
* **測試方法**：
  在 FreeRTOS 的預設任務 (Default Task) 循環中，使用定時器或 Tick 計數器進行分頻翻轉，形成 **1 Hz 的系統溫和心跳** (0.5 秒亮、0.5 秒暗)。
* **驗證指標**：
  * 板載 LED_SYS 呈現穩定的 1 Hz 呼吸閃爍。
  * 即使在高頻感測器採樣下，心跳燈亦無卡頓或頻率抖動，證明 RTOS 排程與空閒處理正常。

### 2. USART2 序列輸出與 printf 重定向
* **硬體引腳**：`PA2` (TX), `PA3` (RX)。
* **測試方法**：
  在 `main.c` 的 `USER CODE BEGIN 4` 區塊重定向 `_write` 函數，將標準 `printf` 輸出重定向至 `USART2` (HAL_UART_Transmit 阻塞傳輸)。
* **測試要點 (避免浮點解算開銷)**：
  在嵌入式系統中，為避免 `printf` 的 `%f` 嚴重消耗堆疊與 CPU 時間，所有感測器數據均在乘上固定倍率後轉為**整數**輸出：
  * 加速度數據：乘上 `1000` (單位：milli-g)
  * 高度與相對高度：乘上 `100` (單位：cm)
  * 溫度數據：乘上 `100` (單位：°C * 100)
* **驗證指標**：
  * 使用序列除錯助手 (如 `screen`, `PuTTY` 或 CubeIDE 內建 Terminal) 以波特率 **115200 bps** 連接，能流暢接收到實時遙測數據字串。

### 3. 獨立看門狗 (IWDG) 超時防禦測試
* **測試方法**：
  配置 IWDG 超時時間為約 2 秒。在 FreeRTOS 主任務無窮迴圈的頂端定期呼叫 `HAL_IWDG_Refresh(&hiwdg)`。
* **驗證方法**：
  * 在正常運作下，系統能持續輸出數據，無自動重啟現象。
  * 故意在程式碼中加入大於 2 秒的阻塞延遲（例如 `HAL_Delay(3000)`），驗證系統是否能成功被看門狗硬體重置 (Reset)，並在串口輸出重新啟動的日誌。

## 📡 二、 傳感器高頻中斷與雙緩衝測試方法

為了發揮感測器的最高物理採樣率，同時避免頻繁的上下文切換 (Context Switch) 拖垮 RTOS 任務，系統採用了**硬體定時器中斷 + Ping-Pong 雙緩衝區 (Double Buffer)** 機制。

### 1. ADXL375 高-G 加速度計高頻測試 (SPI1)
* **硬體引腳**：`PC4` (CS)，共用 SPI1。
* **測試方法**：
  1. 使用 **TIM3** 觸發 **3200 Hz** 硬體中斷。
  2. 在 TIM3 中斷服務程式中，直接透過 SPI1 讀取 ADXL375 的 3 軸加速度原始數據 (7 位元組，耗時約 5.6µs)，並寫入雙緩衝區 `ADXL_Sample_t g_adxl_batches[2][32]` (每 32 筆為一組 10ms 數據)。
  3. 當一組緩衝區寫滿時，中斷自動切換至另一組，並設置 `g_adxl_new_batch_ready = 1` 標誌。
  4. FreeRTOS 預設任務在 1ms 輪詢中偵測到該標誌時，一次性取出該批次 32 筆資料（降低任務開銷 96.8%）。
* **SPI1 總線互斥安全鎖**：
  由於 `ADXL375` (定時器中斷讀取) 與 `BMP388` (RTOS 任務層讀取) 共用 SPI1，為防止中斷搶佔導致 SPI 衝突，在任務層讀取 `BMP388` 前後，必須封裝中斷屏蔽鎖：
  ```c
  __HAL_TIM_DISABLE_IT(&htim3, TIM_IT_UPDATE);  // 暫停 TIM3 中斷
  BMP388_ReadData(&hspi1, &baro_data);          // 讀取 BMP388
  __HAL_TIM_ENABLE_IT(&htim3, TIM_IT_UPDATE);   // 恢復 TIM3 中斷
  ```
* **驗證指標**：
  * 實測採樣率精準鎖定在 **3200.0 Hz**。
  * 讀取 BMP388 時，兩者數據均正確無誤，無出現讀寫衝突或 SPI 卡死。

### 2. BMI088 標準姿態 IMU 高頻獨立測試 (SPI2)
* **硬體引腳**：`PB12` (ACCEL_CS)、`PB11` (GYRO_CS)、`PB13` (SCK)，專用 SPI2。
* **測試方法**：
  1. BMI088 加速度計物理極限為 1600 Hz，陀螺儀物理極限為 2000 Hz。
  2. 使用 **TIM6** 觸發 **1600 Hz** 中斷，在其中讀取加速度數據並寫入雙緩衝區 `g_bmi_acc_batches[2][16]`。
  3. 使用 **TIM7** 觸發 **2000 Hz** 中斷，在其中讀取陀螺儀數據並寫入雙緩衝區 `g_bmi_gyro_batches[2][20]`。
  4. 由於 SPI2 專屬於 BMI088，無任何任務層讀寫，因此本架構為 **Zero-Lock 免鎖設計**，兩個定時器中斷可安全併行，無需任何中斷屏蔽鎖。
* **驗證指標**：
  * 加速度計採樣率精準達到 **1600.0 Hz**。
  * 陀螺儀採樣率精準達到 **2000.0 Hz**。
  * 即使長時間在高頻中斷下運作，系統依然維持穩定。

### 3. BMP388 氣壓高度計測試 (SPI1)
* **硬體引腳**：`PA4` (CS)，共用 SPI1。
* **測試方法**：
  1. 引入 Bosch 官方 `BMP3 Sensor API` (v2) 以確保補償多項式解算精度。
  2. 為了在 **200 Hz ODR** 週期 (5ms) 下穩定工作，配置為 **1x 氣壓過採樣** 與 **1x 溫度過採樣**，確保轉換時間 $T_{conv} \approx 4.94\text{ ms} < 5\text{ ms}$。
  3. 在 FreeRTOS 預設任務中，每 5ms (200 Hz) 呼叫一次 API 讀取數據。
* **驗證指標**：
  * 高度計輸出氣壓與厘米級相對高度，對手部抬起、吹氣等物理動作有即時靈敏的高度變化反應。

## 💾 三、 儲存外設測試環境與方法

### 1. W25Q128 SPI Flash 測試 (SPI3)
* **硬體引腳**：`PA15` (CS)、`PB3` (SCK)、`PB4` (MISO)、`PB5` (MOSI)。
* **測試方法**：
  1. 主控板透過獨立的 **SPI3** 與 W25Q128 連接。
  2. **啟動自檢**：讀取 W25Q128 的 JEDEC ID。
  3. **讀寫校驗**：
     * 對 Sector 0 進行擦除 (Erase) 操作。
     * 向頁面 (Page) 寫入一段長度為 61 標記的校驗測試字串。
     * 重新讀取寫入的數據，進行 100% 逐字節校驗比對。
* **驗證指標**：
  * JEDEC ID 讀取必須為 **`0xEF4018`**。
  * 寫入的測試數據與讀回的數據完全一致，自檢測試輸出 `[Flash] ✅ Write and Read verification 100% SUCCESS!`。

### 2. SD 卡高速數據日誌測試 (SDIO)
* **硬體引腳**：
  * `PC8` ~ `PC11` (D0 ~ D3), `PC12` (CLK), `PD2` (CMD)。
  * **`PD3`** (Net Label: `SDIO_DET_Pin`) 插入檢測引腳。
* **測試方法**：
  1. **插卡防空保護**：
     * 將 `SDIO_DET_Pin` (PD3) 配置為內部強上拉 **`GPIO_PULLUP`**，防止引腳浮空導致假性插卡檢測。
     * 程式啟動後先進行非阻塞的 FatFs 磁碟掛載 (`f_mount(&SDFatFS, SDPath, 0)`)。
     * 讀取 PD3 的狀態，僅當其為低電平 (GPIO_PIN_RESET，表示有卡插入) 時，才執行建立 CSV 檔案 (`f_open`) 與寫入。若為高電平 (無卡)，則優雅跳過，避免總線讀寫卡死。
  2. **10 秒 100 Hz CSV 高頻寫入**：
     * 偵測到插卡後，在預設任務循環中，每 10ms (100 Hz) 將當前 Tick、BMI088、ADXL375、BMP388 的實時感測器數據打包成 CSV 字串寫入 `HIL_10S.CSV`。
     * 寫入持續整整 10 秒 (10000ms)。
  3. **安全卸載**：
     * 10 秒結束後，立即呼叫 `f_close(&SDFile)` 關閉檔案，並執行 `f_mount(NULL, SDPath, 1)` 強制物理卸載，確保所有緩衝區數據實體寫入磁碟，防止斷電丟失。
* **驗證指標**：
  * **無卡測試**：在未插卡時，系統能流暢啟動，串口遙測輸出無任何卡頓或延遲。
  * **有卡測試**：插入 MicroSD 卡上電，串口輸出 `[SD] ✅ 檢測到 SD 卡已插入！正在建立 HIL_10S.CSV 檔案...`。
  * 10 秒後串口輸出 `[SD] 🎉 10 秒數據寫入結束！檔案已安全關閉並儲存 (HIL_10S.CSV)。`。
  * 將 MicroSD 卡放入電腦中，讀取 `HIL_10S.CSV`，確認 CSV 格式正確，含有完整 1000 筆感測器高頻數據且無資料破損。

## 🏃 四、 測試執行步驟與操作指南

### 步驟 1：硬體準備與接線
1. 將測試用 **STM32F407VGT6** 核心主控板放置於防靜電墊上。
2. 插入一張 FAT32 格式的 MicroSD 卡至板載卡槽中（若要測試防空保護，可先不插）。
3. 使用 USB-TTL 轉接線，將電腦的 RX/TX 分別連接至主控板的 **PA2** (TX) / **PA3** (RX)，並共用 GND。
4. 電腦開啟序列除錯工具，連接對應的 COM Port，參數設定為 `115200 bps, 8-N-1`。
5. 將 ST-LINK 燒錄器連接至板載 SWD 接口。

### 步驟 2：編譯與燒錄
1. 在專案根目錄下，開啟終端機並確認 GCC 工具鏈已在 PATH 中。
2. 執行編譯命令：
   ```bash
   make -j4
   ```
3. 編譯完成後，使用 ST-LINK 程式 (或 STM32CubeProgrammer) 將 `Main_AV_F407/Debug/bigav_407.bin` (或 hex) 燒錄至晶片中。

### 步驟 3：運行與串口觀測
1. 按下板載的 Reset 按鈕重置晶片。
2. 觀察串口輸出，預期應依序顯示以下自檢與日誌：
   * 系統啟動，LED 開始 1 Hz 慢閃。
   * **Flash 自檢**：顯示 JEDEC ID 為 `0xEF4018` 並通過讀寫驗證。
   * **SD 卡檢測**：顯示檢測到插卡，建立 `HIL_10S.CSV`，並開始 100 Hz 高頻寫入。
   * **數據日誌**：10 秒期間，串口會定頻輸出當前感測器（BMI088、ADXL375、BMP388）的遙測數據（乘上倍率後的整數值）。
   * **卸載通知**：10 秒後，輸出日誌結束，SD 卡安全關閉並卸載。
