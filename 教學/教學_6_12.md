# RocketCom 航電系統：從零開始點亮 LED、序列輸出與感測器 SPI 驅動教學

本教學手冊旨在引導成大 ISP 航電組在 **STM32F407VGT6** 主控板上進行軟韌體開發，涵蓋最基本的 GPIO 操作、看門狗餵狗防禦、IMU (BMI088) 驅動、高-G 加速度計 (ADXL375) 與氣壓高度計 (BMP388/390) 的 SPI 驅動與高頻中斷雙緩衝優化。

---

## 🚀 系統硬體與軟體規格簡介

* **核心主控**：STM32F407VGT6 (LQFP100)
* **開發工具**：STM32CubeIDE + STM32CubeMX
* **作業系統**：FreeRTOS (CMSIS_V2)
* **安全防範**：獨立看門狗定時器 (IWDG)

## 🚀 階段一：點亮第一個心跳 LED (GPIO 輸出)

為了驗證核心最小系統正常工作，第一步通常是點亮並翻轉板載的 `LED_SYS`。

### 1. 硬體引腳連接
* **Net Label**: `LED_SYS` -> **`PE2`**
* **控制邏輯**: 低電平 (Low) 點亮，高電平 (High) 熄滅。

### 2. FreeRTOS 任務實作
在預設任務（Default Task）中，我們實作溫和的 **1 Hz 系統心跳 (Heartbeat) 指示**：

```c
void StartDefaultTask(void *argument)
{
  for(;;)
  {
    // 翻轉 PE2 引腳狀態
    HAL_GPIO_TogglePin(LED_SYS_GPIO_Port, LED_SYS_Pin);
    
    // 延遲 500ms (形成亮 0.5s / 暗 0.5s)
    osDelay(500);
  }
}
```

## 🚀 階段二：USART2 序列輸出與 printf 重定向

為了讓航電板能透過 UART 輸出日誌到電腦進行除錯，我們需要重定向 `printf`。

### 1. `printf` 重定向至 USART2 (`PA2`/`PA3`)
在 `main.c` 的 `USER CODE BEGIN 4` 區塊中定義 `_write` 函數：

```c
int _write(int file, char *ptr, int len)
{
  HAL_UART_Transmit(&huart2, (uint8_t*)ptr, len, HAL_MAX_DELAY);
  return len;
}
```

### 2. 避免使用浮點數以節省資源
在嵌入式系統中，`printf` 輸出 `%f` 浮點數非常消耗堆疊空間與 CPU 資源。我們採用乘上倍率轉為整數輸出的黃金法則：
* **加速度數據**：乘上 `1000.0f`，轉為 **milli-g (毫重力)** 整數輸出。
* **高度數據**：乘上 `100.0f`，轉為 **cm (公分)** 整數輸出。
* **溫度數據**：乘上 `100.0f`，轉為 **°C * 100** 整數輸出。

## 🚀 階段三：看門狗 (IWDG) 超時防禦與除錯調整

### 1. 發現問題
在除錯放慢主迴圈時，Live Expressions 中的數據常會莫名歸零並重置，這是因為看門狗（IWDG）超時觸發了硬體 Reset。

### 2. 解決方案
在 FreeRTOS 主任務無窮迴圈的頂端，必須定期呼叫餵狗指令：

```c
  for(;;)
  {
    // 餵狗 (Feed the independent watchdog)
    HAL_IWDG_Refresh(&hiwdg);
    
    // 讀取感測器...
  }
```

## 🚀 階段五：ADXL375 高-G 加速度計 (SPI1 整合)

ADXL375 提供 $\pm 200g$ 的衝擊加速度，掛載於 **SPI1**。
* **HIGHG_CS** -> `PC4`
* 解析度固定為 **49 mg/LSB**。
* 注意事項：因為 PCB 走線軸向相反，實際解算時 X 與 Y 軸數據應取負值以做方向修正。

## 🚀 階段六：BMP388/390 氣壓高度計 (官方 Bosch BMP3 API 完美整合)

BMP388 掛載於 **SPI1**，負責氣壓、溫度與厘米級相對高度。
* **BARO_CS** -> `PA4`

### 1. 為何引進官方 Bosch BMP3 Sensor API (v2)？
Bosch 晶片的校準多項式具有極高精度的量產量化要求（如 $2^{-8}$、$2^{-30}$ 等），手動實現極易因精度損失導致嚴重偏差。我們直接引進官方維護的 BMP3 API（`bmp3.c`、`bmp3_defs.h`），以獲得絕對精確的補償解算。

### 2. 轉換時間與 ODR 限制關係
根據 Datasheet 規定，氣壓計的轉換時間為：
$$T_{conv} = 234\mu s + press\_en \cdot (392\mu s + 2^{osrs\_p} \cdot 2000\mu s) + temp\_en \cdot (313\mu s + 2^{osrs\_t} \cdot 2000\mu s)$$
* 在 **200 Hz ODR (5ms 週期)** 下，我們 must 採用 **1x 氣壓過採樣** 與 **1x 溫度過採樣**：
  $$T_{conv} = 234 + (392 + 2000) + (313 + 2000) = 4939\mu s = 4.94	ext{ ms} < 5	ext{ ms}$$
  這符合 $T_{conv} < T_{ODR}$ 的限制，官方 API 驗證通過，順利啟動。

## 🚀 階段七：飛控級高頻中斷採樣與雙緩衝 (TIM3 + Ping-Pong Buffer ADXL375 實作)

為了解鎖 `ADXL375` 的 3200 Hz 理論最高 ODR，且不干擾 RTOS 任務的正常排程，我們引進了**定時器中斷 + 雙緩衝區（Ping-Pong Buffer）**架構。

### 1. TIM3 3.2 kHz 定時器配置
* **時鐘計算**：TIM3 輸入時鐘為 **$84	ext{ MHz}$**。
* **參數設定**：
  * `Prescaler = 5`（6 分頻，時鐘變為 $14	ext{ MHz}$）。
  * `Period = 4374`（4375 計數）。
  * 頻率計算：
    $$f_{TIM3} = rac{84,000,000}{6 	imes 4375} = \mathbf{3200.0	ext{ Hz}}$$

### 2. Ping-Pong 雙緩衝機制
* 聲明一個雙緩衝陣列：`ADXL_Sample_t g_adxl_batches[2][32];`。
* **中斷阻塞讀取**：TIM3 中斷觸發後，在中斷內直接拉低 `PC4` (HIGHG_CS) 透過 SPI1 讀取 7 位元組原始數據（10 MHz 下僅耗時 **$5.6\mu s$**，僅佔週期的 **$1.8\%$**，對中斷極其安全）。
* **批次打包通知**：當寫入滿 32 筆資料（10 ms）時，自動切換緩衝區，並設置 `g_adxl_new_batch_ready = 1`。
* **RTOS 低頻處理**：FreeRTOS 任務在 1ms 循環內查詢該 Ready 標誌並一次性取出批次數據（可直接進行 3.2 kHz 批次持久化），將 RTOS 切換開銷降低 96.8%（從 3200 Hz 降至 100 Hz）！

### 3. SPI1 總線互斥安全鎖
* **衝突場景**：由於 ADXL375 (定時器中斷讀取) 與 BMP388 (RTOS 任務) 共用 SPI1。如果任務正在讀取 BMP388，TIM3 中斷搶佔總線將導致衝突。
* **解決方案**：在 `main.c` 任務讀取 `BMP388_ReadData` 的前後，封裝中斷屏蔽鎖：
  ```c
  __HAL_TIM_DISABLE_IT(&htim3, TIM_IT_UPDATE);  // 暫停 TIM3 中斷
  BMP388_ReadData(&hspi1, &baro_data);          // 讀取 BMP388
  __HAL_TIM_ENABLE_IT(&htim3, TIM_IT_UPDATE);   // 恢復 TIM3 中斷
  ```
  由於 BMP388 傳輸僅需約 $20\mu s$，TIM3 中斷暫停不會漏失數據（中斷標誌會保持 Pending，重開後立即追補執行），完美保證了 SPI1 匯流排的絕對安全！

## 🚀 階段八：飛控級高頻中斷採樣與雙緩衝 (TIM6 + TIM7 + Ping-Pong Buffer BMI088 1600+2000 Hz 實作)

為了將標準姿態 IMU `BMI088` (加速度計最高實體 ODR=1600 Hz，陀螺儀最高實體 ODR=2000 Hz) 發揮到各自的物理極限，且不讓超高頻傳感器讀取浪費 RTOS 任務排程時間，我們採用了**雙定時器獨立中斷 + 獨立 Ping-Pong 雙緩衝**的高性能解耦架構。

### 1. 雙定時器配置
TIM6 與 TIM7 均掛載在 APB1，輸入時鐘為 **$84	ext{ MHz}$**：
- **TIM6 (加速度計 1600 Hz)**：
  - `Prescaler = 41`（42 分頻，時鐘 2 MHz）。
  - `Period = 1249`（1250 計數）。
  - 頻率：$2,000,000 / 1250 = \mathbf{1600.0	ext{ Hz}}$。
- **TIM7 (陀螺儀 2000 Hz)**：
  - `Prescaler = 41`（42 分頻，時鐘 2 MHz）。
  - `Period = 999`（1000 計數）。
  - 頻率：$2,000,000 / 1000 = \mathbf{2000.0	ext{ Hz}}$。

### 2. 獨立雙緩衝機制
- **加速度計**：`g_bmi_acc_batches[2][16]`（16 筆數據為一個 10ms 批次）。在 TIM6 中斷內呼叫 `BMI088_ReadAccel` 阻塞讀取 6 位元組加速度（耗時僅約 **$5.6\mu s$**），集滿後設置 `g_bmi_acc_new_batch_ready = 1`。
- **陀螺儀**：`g_bmi_gyro_batches[2][20]`（20 筆數據為一個 10ms 批次）。在 TIM7 中斷內呼叫 `BMI088_ReadGyro` 阻塞讀取 6 位元組角速度（耗時僅約 **$5.2\mu s$**），集滿後設置 `g_bmi_gyro_new_batch_ready = 1`。
- **RTOS 數據提取**：在 `StartDefaultTask` 中，每隔 1ms 輪詢兩個標誌，集滿時非阻塞提取各自最後一筆最新採樣數據供 20 Hz 即時遙測傳輸。

### 3. SPI2 零鎖定衝突免除設計 (Zero-Lock Bus Dedication)
- **架構優勢**：因為 **SPI2 匯流排完全專屬於 BMI088 加速度與陀螺儀**，FreeRTOS 的任何其他任務都不會使用 SPI2。
- **安全保障**：我們將 SPI2 的所有操作都封裝進 `TIM6_DAC_IRQHandler` 與 `TIM7_IRQHandler` 中，徹底免除了 RTOS 任務層的呼叫。因此，中斷在讀取時，**絕對不會與任何 FreeRTOS 任務發生 SPI 匯流排搶佔衝突**。
- **性能收益**：完全不需要中斷屏蔽鎖（`__HAL_TIM_DISABLE_IT`）或互斥鎖，以物理硬體定時器優勢實現了零死鎖、零開銷、超低抖動的飛控級數據獲取。

## 🚀 階段十：SD 卡高速 SDIO 數據日誌記錄與插卡防空保護 (FATFS 整合)

為了解鎖飛行數據的實時本地持久化，我們將板載 SD 卡槽（掛載於 SDIO 匯流排）與 FatFs 文件中間件進行整合，實現了啟動後 10 秒 100 Hz CSV 高頻遙測日誌的儲存。同時，為了相容無卡 HIL 測試，我們設計了防空保護引腳邏輯。

### 1. FatFs 與 SDIO 延遲掛載
在 `StartDefaultTask` 啟動時，使用非阻塞的延遲掛載，確保絕不因外部 SDIO 狀態阻塞 FreeRTOS 看門狗（IWDG）：
```c
  // 延遲掛載參數為 0 (不要立即物理掃描卡)，保證非阻塞
  if (f_mount(&SDFatFS, SDPath, 0) == FR_OK) {
      // 順利註冊 FatFs 磁碟區
  }
```

### 2. 物理插卡檢測與無卡防空鎖死保護
* **衝突痛點**：如果 SD 卡未插入，但 `SDIO_DET` 引腳浮空（Floating）讀取為 0，FatFs 的 `f_open` 會進入物理總線讀寫，在 DMA 傳輸中因無響應而永久卡死或超時，導致串口遙測無法啟動，甚至看門狗復位。
* **解決方案**：
  1. 將 `SDIO_DET_Pin` (PD3) 引腳配置由 `GPIO_NOPULL` 修改為 **`GPIO_PULLUP`**，確保當沒有 SD 卡插入時，PD3 被內部強拉為高電平 (1)，而有卡插入時被金屬彈片下拉為低電平 (0)。
  2. 在掛載後，檢測 `SDIO_DET` 狀態。只有為低電平時才執行 `f_open` 建立日誌，否則優雅跳過：
```c
  if (HAL_GPIO_ReadPin(SDIO_DET_GPIO_Port, SDIO_DET_Pin) == GPIO_PIN_RESET) {
      printf("[SD] ✅ 檢測到 SD 卡已插入！正在建立 HIL_10S.CSV 檔案...
");
      if (f_open(&SDFile, "HIL_10S.CSV", FA_CREATE_ALWAYS | FA_WRITE) == FR_OK) {
          sd_logging_active = 1;
          sd_log_start_tick = HAL_GetTick();
          // 寫入 CSV Header
          char header[] = "tick_ms,bmi_ax,bmi_ay,bmi_az,adxl_ax,adxl_ay,adxl_az,temp,press,alt
";
          UINT bw;
          f_write(&SDFile, header, sizeof(header) - 1, &bw);
      }
  } else {
      printf("[SD] ⚠️ 未檢測到 SD 卡插入（SDIO_DET 為高電平），跳過 10 秒日誌寫入。
");
  }
```

### 3. 10 秒 100 Hz CSV 高頻寫入與安全卸載
在 Default Task 循環中，若 `sd_logging_active` 裝態啟用，利用 `tick` 計數器進行 100 Hz (10ms) 分頻寫入：
```c
  if (sd_logging_active) {
      uint32_t current_elapsed = HAL_GetTick() - sd_log_start_tick;
      if (current_elapsed < 10000U) { // 限制為 10 秒
          if (tick % 10 == 0) { // 100 Hz (10ms) 寫入一次
              char csv_buf[128];
              int len = snprintf(csv_buf, sizeof(csv_buf), "%u,%d,%d,%d,%d,%d,%d,%d,%d,%d\r\n", ...);
              UINT bytes_written;
              f_write(&SDFile, csv_buf, len, &bytes_written);
          }
      } else {
          // 滿 10 秒，關閉並儲存檔案，卸載 FatFs，徹底將緩衝寫回磁碟，避免數據丟失
          f_close(&SDFile);
          f_mount(NULL, SDPath, 1); // 強制卸載 (1)
          sd_logging_active = 0;
          printf("[SD] 🎉 10 秒數據寫入結束！檔案已安全關閉並儲存 (HIL_10S.CSV)。\r\n");
      }
  }
```
這套架構在保護了 HIL 測試的流暢度的同時，也保證了實體發射時黑盒子數據日誌的極高可靠性與實時性能！
